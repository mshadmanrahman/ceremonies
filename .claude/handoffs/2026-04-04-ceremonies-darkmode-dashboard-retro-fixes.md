### 4. Retro Dogfood Fixes (30 cards / 14 participants)

Full bug report generated by agent. Six fixes shipped:

#### Fix A: Scatter overlap (CRITICAL)
**File:** `src/lib/state-machines/retro.ts` → `scatterCardPositions`  
**Problem:** With 30 cards, computed cols=7 but cellW=140px < cardWidth=180px → guaranteed overlap  
**Fix:** `maxCols = floor(usableW / (cardWidth + 8))` caps columns so cellW ≥ cardWidth

#### Fix B: Rename persistence (HIGH)
**Files:** `src/lib/state-machines/retro.ts`  
**Problem:** `computeProximityGroups` replaces `state.groups` entirely on every `MOVE_CARD_POSITION`, wiping user-renamed labels  
**Fix:** Added `renamedLabels: Record<string, string>` to `RetroState` (key = fingerprint of sorted card IDs). `RENAME_GROUP` saves to fingerprint map. `MOVE_CARD_POSITION` restores labels from map after each proximity recalc.  
**Migration:** `party/retro.ts` onStart migrates old state with `renamedLabels: saved.renamedLabels ?? {}`

#### Fix C: Voting deadlock (HIGH)
**File:** `src/components/retro/voting-phase.tsx`  
**Problem:** "Start discussion" button only shows when ALL participants voted. One person leaving = permanent deadlock.  
**Fix:** Added "Force reveal & start discussion" underline button for facilitator (always visible when not all votes in). Subtle, not prominent, but always reachable.

#### Fix D: Reconnect continuity (HIGH)
**Files:** `src/hooks/use-retro-room.ts`, `party/retro.ts`  
**Problem:** Each WebSocket connection generates a new `anonymousId`. Refresh = can no longer edit/delete your own cards.  
**Fix:**
- Client: `localStorage.getItem('retro-anon-{roomId}')` — creates and caches on first join, reuses on reconnect. Sent as `anonId` query param.
- Server: `url.searchParams.get("anonId") || generateId()` — reuses client-provided ID.

#### Fix E: Canvas label clipping (MEDIUM)
**File:** `src/components/retro/grouping-phase.tsx`  
**Problem:** Group boundary label at `-top-3` (12px above canvas boundary) clipped by `overflow-hidden`  
**Fix:** Moved label to `top-1.5` — inside the boundary, always visible.

#### Fix F: Solo cards = groups of 1
**File:** `src/lib/state-machines/retro.ts` → `computeProximityGroups`  
**Problem:** `if (members.length < 2) continue` — lone cards invisible during grouping, silently auto-grouped at voting  
**Fix:** Removed the guard. Every card (solo or grouped) gets a boundary + rename label during grouping phase.

## Architecture Notes

### Internal API Secret Pattern
```
PartyKit server → X-Internal-Secret header → Next.js API route
```
The route checks header first, falls back to Clerk auth. `createdBy` comes from request body for internal calls. Pattern can be reused for any future PartyKit→Next.js server-to-server call.

### Rename Fingerprint Pattern
```ts
groupFingerprint(cardIds) = [...cardIds].sort().join(',')
// stored in state.renamedLabels[fingerprint] = label
// restored after every computeProximityGroups call
```
Label survives as long as group membership doesn't change. Adding/removing cards from a group resets it to auto-generated label (intentional — group changed fundamentally).

### AnonymousId Continuity Pattern
```
localStorage('retro-anon-{roomId}') → query param 'anonId' → PartyKit reads + stores in conn.state
```
Survives page refresh, network flicker, browser close+reopen. Does NOT survive clearing localStorage.
