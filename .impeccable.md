# Ceremonies — Design Context

## Target Audience
Engineering teams (5-12 people) running agile ceremonies: sprint estimation and retrospectives. Primary user is the PM or scrum master facilitating sessions with a distributed team. Secondary users are developers participating in real-time.

## Use Cases
- Facilitating estimation poker with visible votes and card reveals
- Running structured retrospectives with anonymous writing and collaborative grouping
- Reviewing "The Haunting" (last retro's unfinished action items)
- Tracking team health over sprints (sentiment, accuracy, facilitation rotation)

## Brand Personality & Tone
**Playful maximalism meets ritual precision.** Ceremonies are team rituals, and rituals deserve energy, personality, and joy. This is NOT another gray SaaS dashboard. Think: a game night hosted by a design studio.

Tone spectrum: 70% playful, 30% professional. Fun but never childish. Bold but never chaotic.

## Design Inspiration
Primary: **planetono.space** — the gold standard for this project's aesthetic.

### What to steal from Planetono
- **Display typography**: Tight negative letter-spacing (-.06em), bold weight extremes, uppercase transforms for headings
- **Color philosophy**: Bold primary color + black + off-white. Hard 3px borders, not soft shadows
- **Motion system**: Cubic-bezier(.25, 1, .5, 1) easing everywhere (bouncy but controlled). Jelly animations on interactive elements. Staggered word-by-word entrance animations
- **Character-driven storytelling**: Each ceremony type gets a mascot/personality (like Planetono's Major Paws, Cluck Rogers). Estimation Room and Retro Room should feel like distinct "worlds"
- **Full-screen hero sections**: 100svh with grid-area based layouts
- **Asymmetric mobile nav**: Floating bubble (border-radius: 24px), not a hamburger menu
- **Decorative elements**: Circular gradients, star accents, SVG strokes with non-scaling stroke

### Secondary inspiration
- **Reckless by Displaay**: High-contrast editorial serif energy for hero headlines
- **Good Fella (Awwwards)**: Clean hierarchy with bold orange accent punches
- **Shopify Editions**: Premium product showcase, modular card sections, generous whitespace

## Typography Direction
- **Display/Headings**: A distinctive display face. Not Inter, not Roboto, not Space Grotesk. Consider: Futura Passata, Syne, Cabinet Grotesk, or a high-contrast serif like Instrument Serif/Fraunces for ceremony names
- **Body**: A readable companion. Poppins, Nunito, or similar rounded-friendly body text
- **Monospace**: For estimation numbers, vote counts, timer displays. Geist Mono or JetBrains Mono
- **Weight pairing**: Extreme contrast. 100-200 for labels vs 800-900 for headings
- **Letter-spacing**: Tight (-.02em to -.06em) on display, normal on body

## Color System
- **Primary**: A bold, saturated ceremony color (think warm red, electric blue, or deep violet). NOT corporate blue, NOT startup purple gradient
- **Secondary**: Black (#111 or #0a0a0a) for text and borders
- **Surface**: Off-white or warm cream (not pure #fff, too clinical)
- **Borders**: 2-3px solid black. Hard edges, not blurred shadows
- **Accent per ceremony**: Estimation Room gets its own accent, Retro Room gets another. Team Dashboard gets a third. Each room = its own world
- **Dark mode**: Yes, and it should be the default for the hosted version

## Motion & Animation
- **Easing**: cubic-bezier(.25, 1, .5, 1) as the global default. Bouncy, intentional
- **Card reveals**: Estimation cards flip with spring physics, not basic opacity fades
- **Vote animations**: Confetti or particle burst when consensus is reached
- **The Haunting**: Ghost-float entrance animation for returning action items
- **Typing indicators in retro**: Explicitly ABSENT (true anonymity). But grouped cards should animate into clusters
- **Timer**: Pulsing ring animation during time-boxed discussions
- **Page transitions**: Staggered element entrance, not page-level fades

## Layout Principles
- **Full-screen ceremony rooms**: The estimation and retro views are immersive, not dashboard-y
- **Card-based for dashboards**: Team Dashboard uses modular cards with clear visual separation
- **Generous whitespace**: Let the bold typography breathe
- **Responsive strategy**: Mobile-first. Estimation works on phones (participants vote from their device)
- **Grid**: CSS Grid for ceremony layouts, not flexbox hacks

## Mascots & Character System
Each ceremony room should have personality:
- **Estimation Room**: A character that embodies speed/decisiveness (think: a focused owl, a calculating cat)
- **Retro Room**: A character that embodies reflection/honesty (think: a wise bear, a thoughtful ghost)
- **The Haunting**: Ghost-themed returning action items. Items that survived two retros get "angry ghost" treatment
- Style: Flat illustration with bold outlines, matching the 3px border aesthetic. NOT 3D, NOT realistic, NOT emoji

## Anti-Patterns (What NOT to Do)
- NO generic SaaS dashboard aesthetic (gray sidebar, white cards, Inter font)
- NO purple-to-blue gradients (the "AI startup" look)
- NO subtle gray-on-slightly-less-gray color choices
- NO basic CSS transitions (ease, linear). Always custom cubic-bezier
- NO stock illustrations or generic SVG decorations
- NO hamburger menus on mobile
- NO rounded-everything softness (use sharp corners + selective rounding)
- NO "minimal" that's actually just "empty and boring"
- NO loading spinners (use skeleton screens with personality)

## Technical Constraints
- Framework: Next.js 16 + React
- Styling: Tailwind CSS + shadcn/ui (customized heavily)
- Real-time: PartyKit for WebSocket connections
- Font loading: next/font with variable fonts, zero layout shift
- Performance: Core Web Vitals green. Animations use transform/opacity only (GPU-composited)
- Accessibility: WCAG AA minimum. Bold colors must pass contrast. Animations respect prefers-reduced-motion
